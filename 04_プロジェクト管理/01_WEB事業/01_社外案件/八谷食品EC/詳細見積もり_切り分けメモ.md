## 2. 見積もり明細の構成案

### 前提となる数字

| 根拠 | 数字 |
|------|------|
| 社内価格表：Shopifyスタンダード | **80万円〜**（商品登録20点まで込み） |
| 商品登録の超過分 | +1万円／20点 → 100点なら超過80点＝**+4万円** |
| 口頭提示済みの概算 | **50万円程度** |
| 月額保守（ライト） | **3万円〜**（価格表。見積もりに必ずセット） |
| アプリ月額（先方負担） | 月1〜2万円目安＋KOMOJU手数料 |

**ギャップの正体：** 価格表基準なら 80万＋商品登録超過4万＋B2B設計（会員ランク×出し分け・アプリ選定設定・KOMOJU連携）で実質 **90万相当** の案件。口頭50万との差 **約40万** をどう扱うかが論点。

### 案A：範囲を絞って50万円に収める（口頭提示を守る案）

| 項目 | 金額 | 備考 |
|------|------|------|
| Shopify基本構築（既製テーマ・デザインカスタマイズ最小） | 30万 | オリジナルデザインなし。テーマ設定＋ロゴ/配色反映まで |
| B2Bアプリ選定・設定（アクセス制御＋卸価格＋ランク2段階） | 12万 | 検証済み構成（Locksmith＋Wholesale系＋顧客タグ）の実装 |
| 送料グループ設定（配送プロファイル） | 3万 | 標準機能。送料マトリクスは先方支給前提 |
| 決済設定（Shopify Payments＋銀行振込＋KOMOJU） | 5万 | KOMOJUはアカウント開設を先方作業とする |
| **合計** | **50万** | |

- **削るもの：** 商品登録100点→**先方作業**（CSVテンプレート支給＋登録レクチャー1回を含む）／会員初期登録→**先方作業 or 招待メール自己登録**／見積書→**下書き注文の標準フローのみ**（PDFアプリは対象外）
- **根拠：** 価格表80万から「デザイン制作」「商品・会員登録」「見積書カスタム」を抜けば、構築コア部分は50万に収まる計算
- **リスク：** 育成枠とはいえ利益は薄い。質問①で「顧客側PDF発行が必要」と返ってきた瞬間に破綻するため、**追加項目の単価表を見積書に併記しておく**こと

### 案B：50万を「基本構築費」と定義し、追加項目で60〜70万にする案（推奨）

| 項目 | 金額 | 備考 |
|------|------|------|
| Shopify基本構築（案Aと同スコープ） | 50万 | 「お打ち合わせ時の概算50万円」＝この部分、と整合を取る |
| 商品登録100点 | 5万 | 価格表ロジック（20点込み＋1万/20点×4）を100点一式に丸め |
| 会員100店舗 初期一括登録（CSV支給前提） | 3万 | タグ付け（通常/プレミアム）含む |
| 見積書PDFアプリ導入・帳票調整（必要な場合のみ） | 5〜8万 | 宛名・印影対応の有無で変動。質問①の回答待ち |
| **合計** | **58〜66万** | 見積書アプリなしなら58万 |

- **根拠：** 口頭の50万を否定せず「基本部分の概算」として筋を通せる。追加分はすべて**作業量が数えられる項目**（商品100点・会員100件・帳票1種）なので説明が立つ
- **提示の仕方：** 「概算でお伝えした50万円は基本構築の費用です。登録代行や見積書PDFはご要望に応じたオプションとして明細化しました」

| 項目 | 金額 |
|------|------|

## 3. リスク・除外事項リスト

### リスク（見積もり前に潰す／見積書に明記する）

| # | リスク | 対応 |
|---|--------|------|
| 1 | **見積書要件の膨張**（顧客側PDF発行・宛名・印影・帳票レイアウト指定） | 工数リスク最上位級。質問①で先に確定。回答前に金額を確定させない。対応する場合は別項目（5〜8万〜）で切り出す |
| 2 | **会員ランク×商品出し分けの設計**（工数リスク①） | アプリ2本（アクセス制御＋卸価格）の組み合わせ検証を含む。テーマとの相性問題が出た場合の調整枠を工数に見込む |
| 3 | **KOMOJU連携テスト**（工数リスク③） | 外部決済のためテスト決済〜返金確認まで必要。KOMOJUの加盟店審査期間は制作スケジュール外（先方のアカウント開設待ちで遅延し得る）と明記 |
| 4 | **下書き注文の1年自動削除仕様** | 動きのない下書き注文は1年で自動削除される。見積もりを長期保管する運用は不可 → 運用ルールとして先方に事前説明し、議事録に残す |
| 5 | **アプリ依存の構成** | Plus不要の代わりに会員機能はサードパーティアプリ依存。アプリ側の仕様変更・値上げ・サービス終了は保証対象外と見積書に明記（→月額保守で吸収する提案材料にもなる） |
| 6 | **コンビニ払いの手数料体系** | Shopify Paymentsと別体系。手数料率を先方が把握しないまま開店すると粗利計算が狂う → 質問②で先方負担を確定 |

### 除外事項（見積書の「対象外」欄にそのまま使う）

- 商品撮影・原稿（商品説明文）ライティング一式 … **先方対応**（MTG合意済み）
- Shopifyプラン月額・アプリ月額（目安 月1〜2万円）・KOMOJU等の決済手数料 … **先方負担**（質問②で確認）
- オリジナルデザイン制作（既製テーマのカスタマイズ範囲を超えるもの）
- KOMOJU加盟店契約の締結・審査対応（開設サポートは範囲内、契約主体は先方）
- 商品データの整理・整形（整った形式でのご支給が前提。整形が必要な場合は別途）※質問③の回答で確定
- 会員一覧の作成（CSV支給が前提）※質問④の回答で確定
- 公開後の機能追加・アプリ入れ替え対応（→月額保守プランで対応）
- 基幹システム・会計ソフト等との連携
